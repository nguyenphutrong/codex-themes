# Codex Themes

Codex Themes is a community gallery for Codex editor themes. It lets people browse curated theme JSON files, preview them in a Codex-style interface, copy the `codex-theme-v1:` payload, and prepare new submissions for a pull request.

## What is in this repo

- A TanStack Start app that renders the gallery and contribution docs
- A static theme library stored in `data/themes/*.json`
- A `/submit` route that validates and exports new theme payloads
- Tests, linting, and Netlify deployment support for the public site

## Local development

This project uses Bun.

```bash
bun install
bun --bun run dev
```

Open `http://localhost:3000`.

## Quality checks

```bash
bun --bun run check
bun --bun run test
bun --bun run build
```

`check` runs Biome, `test` runs Vitest, and `build` produces the production bundle.

## Project structure

- `data/themes/`: theme records used by the gallery
- `src/routes/`: TanStack Start routes for the gallery, docs, and submit flow
- `src/lib/`: theme parsing, validation, and utility helpers
- `src/components/`: gallery, preview, and shared UI components
- `CONTRIBUTING.md`: contributor-facing rules for adding a theme

## Contributing themes

Each theme is a JSON wrapper around a valid `codex-theme-v1` payload plus discovery metadata such as slug, author, description, tags, and creation date.

Start here:

- Read [CONTRIBUTING.md](./CONTRIBUTING.md)
- Duplicate a nearby file in `data/themes/`
- Run the quality checks above before opening a pull request

The `/submit` route is an export helper for contributors. It does not publish themes automatically.

## Deploying to Netlify

This repo is configured for Netlify using the official TanStack Start Netlify Vite plugin and `netlify.toml`.

If you connect the repository in Netlify, the build settings are:

```toml
[build]
  command = "vite build"
  publish = "dist/client"
```

Netlify will use the server bundle generated by TanStack Start for SSR routes.
